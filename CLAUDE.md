## プロジェクト概要

Android モバイルアプリケーションのプロジェクト。**パブリックリポジトリ**のため、コミット・プッシュされた内容はすべて公開される。

## Security Rules

### Git Commit / Push 時のチェック項目

コミットやプッシュを行う前に、以下の秘匿情報・機密情報が含まれていないことを必ず確認すること:

- API キー、シークレットキー、トークン（例: `AIza...`, `sk-...`, `ghp_...`）
- パスワード、認証情報
- `.env` ファイル、`local.properties` のハードコードされた秘匿値
- Firebase / Google Services の設定ファイル（`google-services.json`）
- キーストアファイル（`.jks`, `.keystore`）および署名情報
- AWS / GCP / Azure などのクラウド認証情報
- プライベート SSH 鍵
- 個人情報（メールアドレス、電話番号、住所など）
- 社内 URL、内部サーバーのホスト名・IP アドレス

### 対応方針

- 秘匿情報は環境変数または `local.properties` 経由で注入し、コードにハードコードしない
- `.gitignore` に秘匿ファイルが含まれていることを確認する
- コミット前に `git diff --staged` で差分を確認し、秘匿情報の混入がないかチェックする
- 万が一秘匿情報がコミットされた場合は、履歴からの除去（`git filter-branch` 等）と該当キーの無効化・ローテーションを行う
